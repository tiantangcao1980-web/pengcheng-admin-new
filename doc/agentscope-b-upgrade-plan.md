# AgentScope B 方案升级路线（全面升级）

## 目标
- 从“单 AI 服务调用”升级为“多智能体协同编排”架构。
- 让 AI 能力成为独立编排层，业务服务（pengcheng-core）作为稳定工具层。
- 打通会话记忆、知识库、审批/报表等多场景，支持后续 AgentScope Runtime 深度接入。

## 分阶段落地

### Phase 0: 安全与基线
- 移除配置中的明文密钥/明文数据库密码，改为环境变量注入。
- 统一 Spring AI / Spring AI Alibaba 依赖版本，避免运行期不兼容。
- 建立 AI 性能与可用性基线（超时、错误率、平均响应时间）。

### Phase 1: 编排层抽象（已落地）
- 新增 `com.pengcheng.ai.orchestration`：
  - `AgentRouteService`：意图路由（报表/知识库/文案/通用）。
  - `AgentOrchestratorService`：统一编排入口。
  - `ConversationMemoryService`：Redis 会话记忆。
- App 与 Admin AI 入口统一接入编排层：
  - `/app/ai/chat`
  - `/admin/ai/chat`、`/admin/ai/chat/stream`
- 引入编排线程池与可配置超时。

### Phase 2: 工具体系标准化（已落地）
- 将报表、知识库、文案、审批、客户查询封装为统一 Tool 契约。
- 增加工具权限域（按角色、数据范围、项目范围）。
- 增加工具审计日志（输入摘要、输出摘要、耗时、调用链）。
- 新增审批/客户查询路由与工具实现，统一纳入编排与审计链路。

### Phase 3: AgentScope Runtime 深度接入
- 用 AgentScope Runtime 替换当前规则路由：
  - Router Agent
  - Report Agent
  - Knowledge Agent
  - Approval Agent
- 接入沙箱/MCP，隔离高风险工具执行。
- 引入多 Agent 协作模式（Sequential/Parallel/Loop）。
- 已完成第 1 步：新增可插拔路由层（rule/agent/hybrid），支持 Router Agent 失败自动回退规则路由。
- 已完成第 2 步：统一 Tool 结构化输出协议（content + structuredData），打通 App/Admin 接口与前端消费链路。
- 已完成第 3 步（基础版）：新增 AgentScope Runtime 路由适配器与组合路由客户端，支持 Runtime 优先、失败自动回退 ChatClient。
- 已完成第 3 步（增强）：固定 Runtime 路由协议（request/response v1）并保留 legacy 解析兼容。

### Phase 4: 生产级优化
- 引入会话压缩（context compaction）与长期记忆策略。
- 引入人类审核节点（高风险审批建议必须 HITL）。
- 建立 A/B 实验和离线评估集，按场景持续优化路由与提示词。
- 已完成第 1 步（HITL 增量）：审批智能体新增高风险识别与 `requiresHumanReview` 结构化输出，按金额阈值标记“必须人工复核”。
- 已完成第 2 步（基础版）：新增会话压缩与摘要长期记忆，编排层将“历史摘要 + 最近窗口”注入 Chat 提示词。
- 已完成第 3 步（实验与评估）：新增路由 A/B 分流（按会话稳定分桶）与提示词 A/B 开关，补充离线路由评估集（JSONL）和准确率基线测试。
- 已完成第 4 步（保护与观测）：新增实验自动回滚保护（失败阈值 + 冷却期），并在编排调用链中补充 `routeMode` / `routeExp` 便于线上审计与排障。
- 已完成第 5 步（指标接口）：新增实验状态与路由实验聚合统计接口（按日期/场景/实验组），支持成功率与平均耗时观测。
- 已完成第 6 步（提示词观测）：调用链新增 `promptExp` / `promptVersion` 标签，并新增提示词实验聚合统计接口。
- 已完成第 7 步（运维控制）：新增实验配置快照与保护开关运维接口，支持手动重置冷却状态、手动封禁 route/prompt 实验分支。
- 已完成第 8 步（配置中心化）：实验参数支持在线更新并持久化到 `sys_config_group`（`aiExperiment`），并在应用启动时自动回放到运行时配置。
- 已完成第 9 步（变更审计与回滚）：新增实验配置审计日志（记录前后配置/操作者/来源），并支持按审计记录一键回滚。
- 已完成第 10 步（治理控制台）：`pengcheng-ui` 新增 AI 实验治理页面，打通配置更新/重载、保护状态重置与封禁、路由与提示词统计查询、配置审计列表与一键回滚。
- 已完成第 11 步（异常告警与抑制）：新增实验异常告警日志（自动回滚/手动封禁），支持同键抑制窗口、防告警风暴、抑制状态重置，并在治理控制台提供告警查询。
- 已完成第 12 步（告警通知打通）：实验异常告警在未被抑制时发布事件，自动桥接为站内系统通知并通过 WebSocket 广播给在线用户。
- 已完成第 13 步（邮件告警通道）：新增实验异常邮件告警开关与收件人配置，告警事件监听器支持按收件人列表发送邮件（失败不影响站内通知）。
- 已完成第 14 步（Webhook 告警通道）：新增实验异常 Webhook 告警开关、地址列表和超时配置，告警事件监听器支持 JSON 负载推送（失败不影响站内通知/邮件）。
- 已完成第 15 步（渠道策略化）：新增 critical/warning 级别渠道配置（notice/email/webhook），告警监听器按级别路由分发并在渠道全部不可用时回退站内通知。
- 已完成第 16 步（投递日志与重试）：新增告警投递日志表，邮件/Webhook 投递结果落库，支持失败后自动重试与手动触发重试，并在治理控制台提供投递日志查询。
- 已完成第 17 步（死信与重放）：投递耗尽重试后进入 dead 死信队列并触发升级站内告警，支持死信列表查询与按投递记录 ID 一键重放。
- 已完成第 18 步（死信处置闭环）：新增投递汇总统计（success/pending/dead/closed/failed）、单条/批量死信关闭、批量重放接口，并在治理控制台提供死信勾选批量处置能力。
- 已完成第 19 步（健康度评估）：新增投递健康度接口（HEALTHY/WARNING/CRITICAL），按死信率/待重试率阈值评估告警通道健康并在治理控制台展示处置建议。
- 已完成第 20 步（健康巡检自动化）：新增投递健康度定时巡检任务与手动巡检接口，CRITICAL 状态下自动升级站内通知并通过 WebSocket 广播，支持冷却抑制防风暴。
- 已完成第 21 步（巡检参数配置中心化）：健康巡检开关/窗口天数/阈值/升级冷却参数接入配置中心快照与在线更新，并在治理控制台支持可视化编辑生效。
- 已完成第 22 步（健康状态迁移告警）：巡检结果新增前后级别与变更标记，状态切换到 WARNING/HEALTHY 时分别触发预警/恢复通知，形成“预警-告警-恢复”闭环。
- 已完成第 23 步（巡检历史与趋势留痕）：新增健康巡检日志表并将每次巡检结果落库，提供历史查询与趋势汇总接口，治理控制台新增历史列表与趋势统计面板。
- 已完成第 24 步（巡检日志治理）：新增健康巡检日志存储概览与过期清理能力，支持手动触发清理与调度自动清理，防止日志表无界增长。
- 已完成第 25 步（阈值建议闭环）：基于历史巡检样本新增健康阈值建议接口（dead/pending），治理控制台支持生成建议并一键应用到实验配置。
- 已完成第 26 步（按日趋势序列）：新增健康巡检按日聚合趋势接口（级别分布/平均与峰值比率），治理控制台新增日趋势表用于观察波动与回归。
- 已完成第 27 步（高风险日期识别）：基于按日趋势计算风险分值并输出高风险日期 TopN，治理控制台新增风险日表用于快速定位异常时间段。
- 已完成第 28 步（风险日下钻联动）：健康巡检日志查询支持按日期过滤，治理控制台支持点击风险日一键下钻到当日巡检日志并快速排障。
- 已完成第 29 步（风险分布汇总）：新增高/中/低风险天数与最高风险日汇总接口，治理控制台展示风险分布概览与高风险日期列表。
- 已完成第 30 步（治理建议自动生成）：聚合健康评估、风险分布与阈值建议生成治理建议接口，治理控制台支持一键生成并展示关键发现与处置动作清单。
- 已完成第 31 步（治理报告导出）：新增治理报告文本接口，聚合健康现状/风险分布/阈值建议/处置动作，治理控制台支持一键生成并展示报告正文。

- 控制器升级：
  - `pengcheng-api/pengcheng-app-api/.../AppAiController.java`
  - `pengcheng-api/pengcheng-admin-api/.../AiChatController.java`
  - `pengcheng-api/pengcheng-admin-api/.../AiKnowledgeController.java`
