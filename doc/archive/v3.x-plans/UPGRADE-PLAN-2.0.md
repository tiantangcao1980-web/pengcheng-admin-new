# 2.0 阶段升级计划：智能化办公与销售协作平台

## 1. 核心目标
将传统的后台管理系统升级为类似钉钉/飞书的“智能化办公与销售协作平台”，重点解决功能冗余、体验割裂问题，并深度集成 AI 能力。

## 2. 架构与功能重构

### 2.1 菜单与导航优化 ("消肿")
*   **统一入口**：移除分散的“文件管理”、“消息中心”等重复入口，统一收敛至【协作办公】和【房产业务】两大板块。
*   **房产业务集成**：将原本独立的房产业务模块（考勤、报销、项目管理等）融入协作办公流，实现“业务即协作”。

### 2.2 智能表格 (Smart Table) 2.0
*   **对标产品**：Airtable / 飞书多维表格
*   **核心特性**：
    *   **动态列类型**：支持文本、数字、单选/多选、人员、日期、附件、关联引用。
    *   **多视图切换**：表格视图、看板视图、画廊视图、甘特图（预留）。
    *   **行内编辑**：像 Excel 一样直接点击单元格编辑。

### 2.3 在线文档与文件管理
*   **深度集成**：不再是简单的文件列表，支持文件夹层级、拖拽移动。
*   **在线编辑**：集成 Office Online / Markdown 编辑器，支持版本历史。

### 2.4 智能销售 OA
*   **AI 销售助手**：在客户管理、房产项目页面植入 AI Copilot，提供客户画像分析、话术建议、风险预警。
*   **日程与会议分离**：
    *   **会议**：专注于会议室预定、会议纪要、录音转写。
    *   **日程**：个人时间管理，支持从会议、待办自动同步。

## 3. AI 能力集成 (AgentScope + Spring AI)

### 3.1 架构设计
*   **底层模型层**：通过 Spring AI Alibaba 对接通义千问 (Qwen) 等大模型。
*   **Agent 编排层**：引入 AgentScope 概念，设计多智能体协作模式（如：销售助理 Agent + 法律合规 Agent + 数据分析 Agent）。
*   **管理后台**：
    *   **模型配置**：API Key 管理、参数微调。
    *   **技能 (Skills) 管理**：管理 Agent 可调用的工具（如查库存、发邮件）。
    *   **MCP 服务**：集成 Model Context Protocol，连接外部数据源。

## 4. 执行路线图
1.  **文档更新**：确立新架构标准。
2.  **前端重构**：清理路由，合并菜单，升级 Smart Table 组件。
3.  **AI 后台搭建**：创建 AI 配置与管理页面。
4.  **业务 AI 化**：在销售场景中落地 AI 助手。
5.  **测试与验收**：全链路功能测试。
