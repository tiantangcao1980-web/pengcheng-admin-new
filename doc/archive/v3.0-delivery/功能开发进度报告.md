## 二、API 文档集成详情

### 2.1 依赖配置

**pom.xml 添加**:
```xml
<properties>
    <knife4j.version>4.4.0</knife4j.version>
</properties>

<dependencyManagement>
    <dependency>
        <groupId>com.github.xiaoymin</groupId>
        <artifactId>knife4j-openapi3-jakarta-spring-boot-starter</artifactId>
        <version>${knife4j.version}</version>
    </dependency>
</dependencyManagement>
```

### 2.2 访问地址

| 环境 | 地址 | 说明 |
|------|------|------|
| 开发环境 | http://localhost:8080/api/doc.html | Knife4j 增强 UI |
| 开发环境 | http://localhost:8080/api/swagger-ui.html | 原生 Swagger UI |
| API 文档 | http://localhost:8080/api/v3/api-docs | OpenAPI 3.0 JSON |

### 2.3 配置说明

```yaml
springdoc:
  api-docs:
    enabled: true
    path: /v3/api-docs
  swagger-ui:
    enabled: true
    path: /swagger-ui.html

knife4j:
  enable: true
  setting:
    language: zh_cn
```

## 四、技术债务与改进建议

### 4.1 代码质量改进

| 问题 | 影响范围 | 建议 | 优先级 |
|------|---------|------|--------|
| Controller 方法过长 | 约 15 个类 | 抽取服务层方法 | 中 |
| 前端组件复用度低 | 全局 | 提取通用业务组件 | 中 |
| TypeScript 类型定义不完善 | API 模块 | 完善接口类型 | 低 |
| 单元测试覆盖率低 (约 30%) | 核心业务 | 补充单元测试 | 高 |

### 4.2 架构优化建议

1. **API 版本管理**: 建议引入 `/api/v1/` 前缀，为后续升级预留空间
2. **统一错误码**: 建议建立全局错误码规范，便于前端处理
3. **日志规范**: 建议统一日志格式，集成 ELK/SLS
4. **监控告警**: 建议集成 Prometheus + Grafana
