## 1. 业务背景与行业特殊性

### 1.1 业务画像

| 维度 | 配置 |
|---|---|
| **产品线** | 电动摩托车 + 光伏储能产品 |
| **客户结构** | B2B（经销商/安装商/批发商） + B2C（个人） **双轨** |
| **首发市场** | 北美 + 南美 + 东南亚（3 大市场同时启动） |
| **未来市场** | 中东 + 非洲 + 欧洲 |
| **现有资产** | 独立站已上线，有现有订单 + 客户数据 |
| **核心差异化** | AI 多语言客服 + 光储 IoT 远程监控 |

### 1.2 行业特殊性（影响所有技术选型）

| 维度 | 电摩 | 光储 |
|---|---|---|
| **品类属性** | 含锂电池 → IATA 9 类危险品 | 含锂电池 + 光伏组件大件 |
| **认证矩阵** | EEC / EPA / DOT / 各国 type approval | UL9540 / IEC62619 / VDE / TÜV / CEC |
| **物流要求** | UN38.3 + MSDS + DGD 必备；海运危险品舱位 | 整柜散货 + 海外仓必需 |
| **支付习惯** | B2C: Stripe/PayPal；B2B: T/T 30/70 分期 | B2B: T/T + L/C；定金尾款分期常态 |
| **售后特点** | 整车保修 + 电池单独保修 | **远程 IoT 监控 + OTA + 长期运维** |
| **典型客单价** | $1k–$3k（B2C）/ $50k+（B2B 整柜） | $5k–$50k（户用）/ $100k+（工商业） |
| **复购周期** | 5–8 年 | 10–15 年（但维保/扩容是持续收入） |

### 1.3 关键认知：IoT 是核心护城河

光储 IoT 不是"加个功能"，是产品定位。客户花 $50k 买光储不是买铁盒子，是买"我家/我厂的电力大脑"。

但 **IoT 是重型子系统**（涉及设备端固件、通信协议、时序数据、规则引擎、OTA），不能塞进 V1.0。**B-V1.0 完全不做 IoT，B-V1.1 做数据接收基座，B-V2.0 完整化。**

## 3. 优先级判断（避免范围爆炸）

3 大市场同时首发**不等于**第一天 3 大平台都接。这是常见陷阱。

| 维度 | 优先级 | 理由 |
|---|---|---|
| **现有独立站接入** | ⭐⭐⭐ 第一 | 已有资产，订单/客户**今天就在跑** |
| **B2B 询盘 → 报价 → PI 流程** | ⭐⭐⭐ 第一 | 1 个 B2B 大客户抵 100 个 B2C |
| **多币种 / 多语言 / 多时区** | ⭐⭐⭐ 第一 | 三大市场首发避无可避 |
| **危险品文档自动化** | ⭐⭐⭐ 第一 | 没这个发不出货 |
| **AI 多语言询盘助手** | ⭐⭐⭐ 第一 | owner 真实差异化 |
| **认证证书管理** | ⭐⭐ 第二 | 储备人员前置完成后落地 |
| **第一个外部平台接入** | ⭐⭐ 第二 | 选**最强 1 个**先接，不并行 |
| **IoT 远程监控** | ⭐⭐⭐ 第二期重点 | 单独章节 |
| **第二/三个外部平台** | ⭐ 第三 | V1.2 |
| **海外仓库存对接** | ⭐ 第三 | 量起来再说 |
| **营销自动化（EDM/WhatsApp/复购）** | ⭐ 第四 | 复购数据沉淀够再做 |

## 10. IoT 单独章节（关键决策点）

### 10.1 责任边界（设备端 vs 软件端）

| 层面 | 责任方 | 风险 |
|---|---|---|
| **设备端固件**（MCU 编程 / 通信模组 / 协议栈） | 设备厂家 / 嵌入式工程师 | **软件团队搞不定**，必须有硬件配合 |
| **通信协议**（MQTT / CoAP / 自定义二进制） | 双方约定 | 协议一旦定，固件改一次成本巨大 — **必须 V1.1 之前定稿** |
| **MQTT broker**（EMQX / Mosquitto / AWS IoT） | 软件团队 | 推荐 EMQX 开源版 |
| **设备影子 + 时序数据** | 软件团队 | TimescaleDB（复用 PostgreSQL，省一套基础设施） |
| **OTA 升级** | 双方协作 | 固件 OTA 失败会"砖机"，必须有硬件按键回滚机制 |
| **远程诊断 AI** | 软件团队 + 数据沉淀 | 至少 3 个月真实数据才能训出有用模型 |

### 10.2 IoT 路径

1. **B-V1.0**：完全不做（如果硬塞会拖死整个 V1.0）
2. **B-V1.1**：做"数据接收基座"（MQTT + 设备影子 + 时序库），即使 UI 还没有，先让设备**能上报数据**
3. **B-V2.0**：完整 IoT（监控 UI + 规则告警 + OTA + AI 诊断）
4. **关键依赖**：储备人员第 10 项前置任务（设备端能力盘点）必须在 A 线期间完成

### 10.3 IoT 选型建议

| 路线 | 优势 | 劣势 | 建议 |
|---|---|---|---|
| **自建 EMQX + TimescaleDB** | 数据全控 / 成本可控 / 私有部署支持 | 运维成本（小团队压力大） | **首选** |
| AWS IoT Core | 全托管 / 成熟 | 数据出海贵 / 锁定 AWS | 备选 |
| 阿里云 IoT 平台 | 国内便宜 | 海外节点弱 / 跨境合规问题 | 不推荐 |

**首选自建 EMQX + TimescaleDB 的原因**：光储设备数据**敏感**（涉及客户能源使用习惯，欧盟 GDPR 严管），全托管出问题难追责。

## 13. 真实风险（必须正视）

### 13.1 资金风险
- **3 大市场认证成本**：电池产品认证一个市场 $20k–$50k 起，**3 大市场就是 $60k–$150k 现金支出**，且周期 3–6 个月
- **Twilio / Stripe / WhatsApp Business API 等海外服务费**：按使用量计，预估 B 线月运营 $2k–$5k 起
- **海外仓押金 + 库存资金**：每仓 $50k–$200k 起
- **建议**：先 1 个市场跑通再扩 2 个，资金允许再开第 3 个

### 13.2 团队风险
- **储备人员能力是最大变数**：如果是初级（非中级），整个 B 线节奏 ×1.4
- **owner 是单点**：你既要做架构、又要做 AI、还要做团队管理，过载风险高
- **建议**：A 线期间评估储备人员真实水位，必要时招 1 个有海外电商经验的中级

### 13.3 技术依赖风险
- **IoT 设备端依赖外部**：固件不是软件团队能搞定，必须有可靠的硬件合作方
- **平台 API 不稳定**：TikTok Shop / Amazon API 一年改 2-3 次，要预留 20% 维护工时
- **建议**：每个外部依赖（设备厂家、平台 API、海外仓）签明确 SLA，并准备替代方案

### 13.4 合规风险
- **TikTok Shop / Amazon 对锂电池类目有严苛资质**：审批可能比开发还久，前置工作里要把资质申请同步启动
- **GDPR / CCPA 数据合规**：违规罚款额度大，B-V1.0 必须做用户数据导出/删除接口
- **建议**：聘请专业合规顾问，至少在 B-V1.0 上线前做一次合规审计

### 13.5 业务风险
- **多币种汇率管理**：第一天就要决定结算币种 + 汇率快照策略，否则未来对账噩梦
- **建议**：B-V1.0 Sprint S1 中明确"以 USD 为内部记账币种 + 汇率每日快照"
