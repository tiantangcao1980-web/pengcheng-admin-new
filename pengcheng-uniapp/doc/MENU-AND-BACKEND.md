# 移动端菜单与后台一致性说明

## 当前状态

- **后台管理端（pengcheng-ui）**：左侧主菜单已与「系统管理 → 菜单管理」中的配置**完全一致**，仅固定前置「首页」一项，其余全部由后端按角色返回的菜单树渲染，无重复、无前端硬编码菜单项。
- **移动端（本 Uniapp）**：
  - **tabBar**：固定 4 项（工作台、消息、通讯录、我的），在 `pages.json` 的 `tabBar.list` 中配置。
  - **工作台快捷入口**：在 `pages/workbench/index.vue` 的 `initQuickEntries()` 中写死，如客户报备、客户列表、考勤打卡、审批中心、AI 助手等。
  - **我的页功能列表**：在 `pages/profile/index.vue` 中写死，如个人信息、考勤打卡、我的请假、我的审批等。

## 与后台的对应关系

| 后台菜单管理配置     | 移动端对应入口                     |
|----------------------|------------------------------------|
| 房产业务 → 客户等     | 工作台快捷入口、通讯录、客户相关页 |
| 消息中心             | tabBar「消息」、工作台系统通知     |
| 考勤/审批/申请       | 工作台快捷入口、我的页列表         |
| 智能助手             | 工作台「AI 助手」入口              |

移动端入口目前**未**根据后台「菜单管理」或角色权限做显隐；所有已实现页面对所有登录用户可见。

## 后续可选增强（与后台一致）

若需移动端与后台菜单/权限一致，可考虑：

1. **后端**：为 App 登录接口（如 `/api/app/auth/info`）增加返回「应用端菜单」或「快捷入口列表」（按角色过滤），结构可与后台菜单树类似，或简化为 `{ name, icon, url, permission }` 列表。
2. **移动端**：
   - 工作台：`getWorkbench()` 或新接口返回「快捷入口」列表，用其替换或合并当前 `initQuickEntries()` 的写死数据；按权限/菜单显隐。
   - 我的页：根据同一套「应用端菜单」或权限标识，动态渲染功能列表（考勤、审批、请假等），无权限的入口不展示。

当前实现已满足「固定功能入口」的使用；上述增强为可选，便于与后台菜单管理统一维护。
